### A. 활용 목적
- 사용자가 타 앱 서비스를 사용할 때 쉽게 우리 서비스로 전환할 수 있다
- 플로팅 버튼을 통해 현재 진행 중인 콜의 주요 상태를 확인할 수 있다

| (4) 콜 취소됨            | 콜 수락 후 호출 취소 발생 시 <br>(라이더앱 / 운영툴 발 호출 취소 모두 포함) | - 앱 진입 시<br>- or (1) or (2) or (3) 이벤트 발생 시 <br/>- or 콜 취소 발생 후 3분 초과 시 |
| (5) 그 외 상태 (default) | (1), (2), (3), (4) 플로팅 버튼 생성 조건 불만족 시           | (1), (2), (3), (4) 이벤트 발생 시                            |



#### 네비게이션 화면 랜딩 로직 

- 콜 수락 ~ 결제 완료 시, 출발지 POI 또는 도착지 POI로 인앱 네비게이션으로 길안내를 제공할 수 있다. 
- 플로팅 버튼을 선택하여 앱에 재진입할 때, (1) 네비게이션 안내 화면 (2) 출발지 / 도착 안내 화면 중 마지막 유지 화면으로 랜딩한다.



### D. 노출 및 위치 조정

- (5) default 상태에서 플로팅 버튼의 위치를 사용자가 원하는 곳으로 조정할 수 있다.
  - 상세 동작은 디자인 가이드 참조 필요 
- 플로팅 버튼의 생성과 앱 실행은 항상 페어로 진행된다. 
  - 앱 실행 중 플로팅 버튼의 임의 삭제는 불가하다.
  - 앱 종료 후 플로팅 버튼의 임의 노출은 불가하다.

### B. 알림 동작

#### 알림의 생성 

- 알림은 해당 이벤트 발생과 동시에 생성된다. 
- 알림은 앱이 백그라운드 실행 시에만 발생하고, 포그라운드 실행 시에는 발생하지 않는다. 
  - 단, 앱이 백그라운드 실행 시 발생한 알림은 포그라운드 전환 시에도 유지된다.

- 알림은 플로팅 버튼의 상태 값과 더불어 기사님께 특정 이벤트 발생을 안내하는 역할을 하며, 일시적으로만 제공된다. 

#### 알림의 삭제 

- 알림은 각 이벤트 별로 유지 기간과 조건이 상이하며, 해당 기간이 만료되거나 조건이 유효하지 않는 즉시 자동 삭제 처리된다. 
- 해당 유지 기간은 기사앱 포그라운드 상태에서 유지 시 노출되는 팝업의 유지 기간과 동일하다. 

| 케이스              | 푸시 유지 기간  | 푸시 유지 조건                                          |
| ------------------- | --------------- | ------------------------------------------------------- |
| 콜 배차 시          | 콜 배차 후 20초 | -                                                       |
| 승객 콜 취소 시     | 콜 취소 후 3분  | -                                                       |
| 출발지 인근 도착 시 | -               | 출발지 인근 200m 도착 시부터 탑승 완료 처리 전까지      |
| 목적지 인근 도착 시 | -               | 도착지 인근 200m 도착 시부터 요금 결제 완료 처리 전까지 |

##### 구성 요소 상세

- 필수 구성 요소는 팝업 게시 시 필수 구성 요소로서 미노출 처리가 불가하다.
- 선택 구성 요소는 운영자가 팝업 게시 목적에 맞게 포함 또는 불포함 처리할 수 있다.  
- 같은 구성 요소는 동일한 텍스트 크기, 색상 등 동일한 스타일을 가진다. 
- 연결 가능한 링크는 웹링크, 딥링크, 공지사항 상세 페이지로 랜딩 가능하다. 
- 팝업 디자인 가이드라인의 최대 height를 초과하는 경우 버튼 영역을 제외한 영역에 대해서 스크롤 가능하다.

| 구성 요소          | 필수/선택 | 커스텀 가능 여부 | 입력 제약 조건                                 |
| ------------------ | --------- | ---------------- | ---------------------------------------------- |
| 타이틀             | 필수      | 가능             | 한영 40자까지 입력 가능                        |
| 본문               | 선택      | 가능             | 한영 500자까지 입력 가능                       |
| 본문 하이라이트    | 선택      | 가능             | 한영 100자까지 입력 가능                       |
| 이미지             | 선택      | 가능             | - width 고정 <br>- height 이미지 비율대로 조정 |
| 링크 레이블        | 선택      | 가능             | 한영 18자                                      |
| 링크 타입          | 선택      | 가능             | 웹링크 / 딥링크 / 공지사항 링크 타입 선택 가능 |
| 링크 url           | 선택      | 가능             | 확인 필요                                      |
| 다시보지 않기 버튼 | 필수      | 불가             | -                                              |
| 닫기 버튼          | 필수      | 불가             | -                                              |

### C. 노출 정책

#### a. 체크 시점
- 운행 준비 화면 랜딩 시점, 공지 팝업 게시 여부 체크 
- 기노출된 공지 팝업 건이라도 공지 기간 내 재노출 가능
  - 다시 보지 않기 선택 시 팝업 닫힘 + 공지 기간 남아 있어도 더 이상 재노출 안함
  - 닫기 선택 시 팝업 닫힘 + 기간 내 재노출

#### b. 노출 대상 

##### 운행 지역
- 사용자의 운행 지역에 해당하는 공지 팝업에 대해서만 노출
  - 구리시에 공지 팝업이 게시되었을 때, 해당 팝업을 구리시 기사님은 확인 가능하나 남양주 시 기사님은 확인 불가
  - 단, 전체 지역에 게시된 공지 팝업은 모든 기사님이 확인 가능
  - 단, 강제 업데이트 팝업과 강제 시스템 팝업은 운행 지역 구분 없이 공통으로 노출

##### 앱 버전 
- 팝업 게시 시점에 설정한 특정 앱 버전 사용자에게만 노출 가능  
- 예) X.X.X 이상 X.X.X 이하

##### 게시 기간
- 팝업 게시 시점에 설정한 게시 기간 내에만 노출 가능
- 예) YYYY.MM.DD. HH:MM ~ YYYY.MM.DD. HH:MM

#### a.  케이스 정의

- 비정상으로 앱이 종료되거나 실수로 앱을 종료시키고 재진입하는 상황 
- getSupply = Ready or On-route or Occupied  && Driver ID 일치 



#### b.  플로우 상세

- 운행 준비 화면 또는 메뉴 하위 화면에서 앱 종료 시, 일괄 운행 준비 화면으로 랜딩한다. 
- 배차 대기, 콜 안내, 호출 취소 화면에서 앱 종료 시, 일괄 배차 대기 화면으로 랜딩한다.
  - Ready 상태에서 이탈 시, Ready 상태로 랜딩 
- 출발지 안내 화면에서 앱 종료 시, 출발지 안내 화면으로 랜딩한다. 
  - On-route 상태에서 이탈 시, On-route 상태로 랜딩 
  - 네비게이션 실행 여부에 상관 없이 항상 출발지 안내 화면으로 랜딩
- 목적지 안내 화면, 요금 입력 화면, 결제 중 화면에서 앱 종료 시, 목적지 안내 화면으로 랜딩한다. 
  - Occupied 상태에서 이탈 시, Occupied 상태로 랜딩
  - 네비게이션 실행 여부에 상관 없이 항상 목적지 안내 화면으로 랜딩
- 요금 결제 완료 후 앱 종료 시, 배차 대기 화면으로 랜딩한다. 
  - Ready 상태에서 이탈 시, Ready 상태로 랜딩 
  
- 앱 종료 후 로그인 세션이 만료되었을 때는 앱 재실행 시 일괄적으로 로그인(휴대폰OTP) 화면으로 랜딩한다.

#### b. 글자 크기 조정 동작 공통

![img](images/05_웹뷰글자크기조정.png)

- 글자 크기는 총 3단계이며 최초 진입 시 1단계 크기로 노출한다.
- 더 크게 정보를 확인하고 싶을 때, + 버튼을 통해 확대 가능하다.
- 더 작게 정보를 확인하고 싶을 때, - 버튼을 통해 축소 가능하다.
- 더 이상 확대, 축소가 불가능할 경우, 불가한 동작 버튼은 딤드 처리되고 토스트가 노출된다. 
- 글자 크기는 서비스에서 제공하는 모든 웹뷰에서 조정할 수 있다.
- 마지막으로 사용자가 설정한 글자 크기 값을 유지한다.
  - 예를 들어, 공지사항에서 3단계 글자 크기로 적용 시, 약관에서도 3단계 글자 크기로 적용

## (9) 안드로이드 Back 키 동작

### A. 공통

| 화면 조건                       | 사용자 액션        | 기대 동작                                     |
| ------------------------------- | ------------------ | --------------------------------------------- |
| 팝업/바텀시트 노출 중 모든 화면 | `Back`             | 팝업/바텀시트 닫힘                            |

| 모든 화면                       | 연속 `Back`+`Back` | 앱 백그라운드 전환 (`Home`키와 동일하게 동작) |

- 별도 화면 별 Back 키 동작의 정의되지 않은 경우, 공통 원칙을 따른다.
