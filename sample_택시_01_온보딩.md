# 01. 온보딩 



## (1) OVERVIEW

![img](images/01_온보딩.png)


- 휴대폰 번호를 통해 `가입 여부`, `심사 여부`를 판단해서 동선을 분기한다. 
- `미가입` & `심사 제출 건 없음` 사용자는 가입 동선을 제공하며, 법인/개인 택시에 따라 입력 정보가 상이하다. 
- `미가입` & `심사 제출 건 있음` 사용자는 `심사 대기 화면`으로 랜딩한다.
  -  `심사 대기 화면` 에서 세션은 임시로 1일 유지된다.
  - 백그라운드 상태에서는 임시 세션이 유지되지만 앱 종료 후 재진입 시에는 세션이 유지되지 않고 휴대폰 OTP 화면으로 랜딩한다. 
  - 세션 만료 후 앱 진입 시 휴대폰 OTP 화면으로 랜딩한다. 
  - 휴대폰 OTP 성공 시 조건에 따라 알맞은 화면으로 랜딩된다. 
- `기가입` 사용자는 `차량 정보 존재 여부`에 따라 동선을 상이하게 제공한다.
  - `기가입` & `차량 정보 존재` 사용자는 `운행 준비`로 랜딩하며, 콜 받기 등 정상적인 서비스 이용이 가능하다. 
  - `기가입` & `차량 정보 미존재` 사용자는 `차량 선택 페이지`로 랜딩하며, 차량 선택을 완료해야 콜 받기 등 정상적인 서비스 이용이 가능하다. 



## (2) 회원 정책 

### A. 기가입 기준 

- 운영툴에서의 `심사 승인`을 기준으로 회원의 가입 여부가 결정된다.
- 운영툴에서 직접 등록된 사용자는 등록 시점부터 `심사 승인`된 것으로 간주한다.

### B. 회원 필수 정보 (🎄v4.9.5)

#### a. 개인 회원 (🎄)

- 개인 회원은 다음 정보를 필수로 가지며, 해당 정보 중 1개라도 null 값이 있는 경우 회원 승인이 불가하다.

  ```
  //<회원 정보>

  - 프로필 사진
  - 실명
  - 생년월일
  - 성별
  - 운행지역 (v4.9.5)
  - 서비스 (v4.9.5)
  - 택시운전자격증명 사진 (택시 운전 자격 번호)* 
  //<차량 정보>
  - 택시 유형 일괄 중형 기선택된 상태 
  - 차량 모델 선택
  - 연료 방식 선택
  - 후면 차량 번호판 사진 (차량 번호)* 
  //<정산 정보>
  - 정산사 선택
  - 사업자등록증 사진 (사업자 등록 번호)*
  
  * 표시 항목은 실제 문서 대조를 통해 진위 여부를 판단하기 위해 사용자 입력 정보와 실제 서버 관리 정보를 상이한 경우이며, 사용자 입력 정보(실제 서버 관리 정보)로 표기
  ```

#### b. 법인 회원 (🎄)

- 법인 회원은 다음 정보를 필수로 가지며, 해당 정보 중 1개라도 null 값이 있는 경우 회원 승인이 불가하다.

  ```
  <회원 정보>
  - (휴대전화번호 프리필, 수정 불가)
  - 프로필 사진
  - 실명
  - 생년월일
  - 성별
  - 운행지역 (v4.9.5)
  - 서비스 (v4.9.5)
  - 운수사
  - 운수사 사번 (4자리 숫자)
  - 택시운전자격증명 사진 (택시 운전 자격 번호)* 
  ```

- 법인 회원은 개인 회원과 다르게 회원 승인 시점 법인의 `정산 정보`가 자동 적용되며, 승인 이후 `차량 정보`를 선택한다. 

- 회원의 키 값은 `택시 운전자 자격 번호`이나 휴대폰 번호가 그에 준하는 회원 판별값으로 활용된다.

### D. 로그인 세션

- 심사 승인 이전 사용자는 회원으로 간주하지 않으므로 로그인 상태를 유지하지 않는다. 
- 심사 승인 이후 최초 로그인 시점 이후 다음과 같은 사용 환경 정보를 서버에 송신한다. 
  - `디바이스 모델`, `운영체제`, `OS버전`, `서비스명`, `앱버전`
- 사용자가 정상적으로 로그인하고 별도 명시적으로 로그아웃하지 않은 경우, 1년 동안 로그인 상태를 유지한다. 



### E. 필수 정보 수정 

- 앱 내에서 필수 정보 수정 동선은 제공하지 않으며, 필수 정보 수정을 위해서는 각 관할 이동지원센터의 운영자 연락을 통해 변경 가능하다. 
- `휴대전화번호`가 변경된 사용자의 경우에도 별도로 로그아웃 처리를 진행하지 않으며, 이후 동일 단말에서 로그아웃 후 재로그인하거나 신규 단말에서 로그인 시에는 새로운 `휴대전화번호`로 OTP 인증 로그인이 필요하다. 
- 그 외 필수 정보가 운영툴을 통해 변경된 사용자의 경우, 별도 로그아웃 없이 메뉴 > 내 정보 진입 시 변경 사항 확인이 가능하다



### F. 멀티 디바이스 로그인

- 한 사용자가 여러 단말에 동시 로그인하는 것을 허용하지 않는다. 
- 한 사용자가 A 단말을 쓰다가 B 단말에서 로그인을 시도하는 경우, B 단말에서의 로그인을 허용하되 A 단말을 자동 로그아웃 처리한다. 

![img](images/01_자동로그아웃_멀티디바이스.png)

### G. 원격 기사 탈퇴

#### a. 발생 경로 

- 운영툴에서 기사를 삭제하는 경우 발생 가능 
- 특히 법인의 경우 기사의 퇴사가 발생하는 경우, 원격으로 기사를 삭제해야 할 필요성 대두
- 개인 / 법인 동작 동일 

#### b. 플로우 상세 

- 앱 진입 시 자동 로그아웃 처리한다.

- 키워드 검색 결과가 없는 경우 검색결과가 없다는 안내 문구를 제공한다. 



##### 검색 결과 동작 상세 

- 문자열 안에 검색 키워드가 여러 번 포함되어있을 경우, 가장 앞의 키워드만 볼드 처리
- 키워드가 문자열에 첫문자에 걸리는 경우에만 검색 결과에 상단에 노출
- 키워드가 문자열에 첫문자에 걸리는 항목이 여러개인 경우, 그 리스트 안에서 오름차순으로 노출
- 키워드가 문자열에 첫 문자에 걸리지 않은 경우에는 오름차순으로 노출
