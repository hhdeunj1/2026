# 2026

AI 에이전트와 함께 GitHub 이슈 작성 워크플로우를 자동화하는 실험 레포입니다.

서비스 기획 문서(MD)를 기반으로 이슈를 생성·관리하는 과정에서,
어디까지 자동화할 수 있는지를 단계적으로 탐색하고 있습니다.

---

## 진행 과정

### 1단계 — 이슈 템플릿 설계
- Feature 이슈 기본 구조 정의 (배경 / Spec / AS-IS / TO-BE)
- Figma 링크, 진입 경로, 연관 MD 등 참조 정보 포함 형식 설계

### 2단계 — 이슈 자동생성 + 템플릿 고도화
- MD 문서를 읽고 이슈를 자동 생성하는 흐름 실험 (feature)
- 템플릿 고도화: 참조 메타와 변경 명세를 `개요` / `Spec`으로 분리
- MD 전체를 분석해 변경의 영향 범위를 자동 검토하는 AI 제안 기능 실험

---

## 앞으로 해볼 것

- **Figma 자동 연결**: 이슈 번호로 기획 Figma 프레임을 찾아 `## Spec` 헤더에 자동 링크
- **MD 변경 감지**: MD 파일 수정 시 관련 이슈에 자동 알림 또는 업데이트
- **이슈 → MD 역방향 반영**: 이슈에서 확정된 스펙을 MD에 다시 반영하는 흐름
- **다른 서비스 도메인으로 확장**: 택시기사앱 외 다른 앱/서비스에도 동일한 워크플로우 적용
